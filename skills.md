## Tool Selection Guide

Use this to pick the right tool before starting any task.

### For Prototyping & UI Generation
| Tool | Best For | Avoid When |
|---|---|---|
| **Figma Make** | Refining existing Figma files with AI; component generation; high-fidelity iteration | Starting from zero with no existing design system |
| **Stitch** | High-fidelity interactive prototypes; user flow simulation; AI-driven interaction design | Quick low-fi wireframe sketches |
| **Figma (manual)** | Design system work; final production-ready screens; handoff | Rapid exploratory ideation |

### For Research & Synthesis
| Tool | Best For | Avoid When |
|---|---|---|
| **Claude** | Brief framing, synthesis, insight clustering, persona refinement, report writing, prompt-based tasks | Real-time transcription |
| **Antigravity (Google)** | Research that lives in Google Drive, Docs, Sheets, or Meet; cross-file synthesis; workspace-integrated workflows | Standalone tasks with no Google data source |
| **Miro / FigJam** | Affinity mapping, journey mapping, collaborative synthesis | Individual solo analysis |

### For Presentations & Communication
| Tool | Best For | Avoid When |
|---|---|---|
| **Gamma.app** | Fast AI-generated pitch decks: draft narrative in Claude first, paste into Gamma | Highly branded, pixel-perfect slide design |
| **Figma** | Designed presentations with custom visual identity | Speed-to-first-draft scenarios |

### For Writing & Content
| Tool | Best For |
|---|---|
| **Claude** | All written deliverables — briefs, reports, pitch narratives, microcopy, prompt writing, handoff docs |

## Skill File Descriptions

### `context.md`
Fill-in-the-blank project context template. Paste at the start of every Claude conversation. Captures project name, user, business goal, current stage, what you need, and tools in use. Highest-frequency file in the system.

### `design.md`
The end-to-end UX/product design process. Covers project kickstart, information architecture, user flows, prototyping strategy, AI ethics/trust checklist, and AI integration at each phase. Start here for any new project.

### `research.md`
Full user research workflow: from writing the brief and recruiting participants, through interviews and surveys, to synthesis, insight clustering, and handoff. Includes AI-augmented steps at every phase.

### `prompts.md`
A ready-to-use prompt library organized by task: briefs, personas, journey maps, Stitch/Figma Make screens, microcopy, flow debugging, synthesis, Antigravity/Google Workspace, and stakeholder communication. Includes a personal log at the bottom for prompts you've tested and want to keep.

### `pitch.md`
Business case and stakeholder communication frameworks. Includes the Value Proposition Canvas, pitch deck structure, ROI measurement, Gamma.app workflow, and how to translate design impact into business language.

### `handoff.md`
Everything needed to close a project phase cleanly. Covers developer handoff checklists, annotation generation, research readout structure, design rationale documentation, stakeholder presentation structure, and Figma file organization standards.

## Skill File Maintenance

When you develop a new reusable prompt, workflow, or framework — add it to the appropriate skill file. Keep entries concrete and action-oriented. Remove anything that hasn't been used in 3+ projects.
